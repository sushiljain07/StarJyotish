# AI Chart Reading Feature — Design Spec

**Date:** 2026-05-31  
**Status:** Approved  
**Scope:** Add a "Reading" tab to the Results page that generates a bilingual AI-powered Vedic chart interpretation using Google Gemini.

---

## Overview

After generating a Kundli chart, users can request a detailed, personalised reading. The reading is generated by Google Gemini (`gemini-2.0-flash`) using structured chart data as context. It is on-demand (not auto-generated) and respects the app's language toggle (English/Hindi).

---

## Architecture

### Components

```
frontend/src/components/ChartReading.jsx   ← new reading UI component
frontend/src/pages/Result.jsx              ← add "reading" to TABS
frontend/src/i18n/en.json                  ← add reading-related keys
frontend/src/i18n/hi.json                  ← Hindi equivalents

backend/services/gemini.py                 ← new: Gemini API client + prompt builder
backend/routes/kundli.py                   ← add POST /kundli/reading endpoint
backend/models/chart_data.py               ← add ReadingRequest + ReadingResponse models
backend/.env                               ← GEMINI_API_KEY (not committed)
backend/requirements.txt                   ← add google-generativeai
```

### Data Flow

1. User views Results page → clicks "Reading" tab
2. Tab renders `ChartReading` component in idle state (button only)
3. User clicks "Generate Reading"
4. Frontend POSTs to `/kundli/reading` with `{ date, time, place, language }`
5. Backend:
   a. Geocodes place (reuses `geocode_place`)
   b. Calculates chart (reuses `calculate_chart`)
   c. Derives planet dignities (exalted, debilitated, own sign, retrograde, combust)
   d. Builds structured prompt
   e. Calls `gemini-2.0-flash` with the prompt
   f. Parses response into 6 named sections
   g. Returns `{ sections: [{ title, icon, content }] }`
6. Frontend renders section cards; shows "Regenerate" button

## Backend

### New endpoint: `POST /kundli/reading`

**Request model (`ReadingRequest`):**
```python
class ReadingRequest(BaseModel):
    date: str       # "YYYY-MM-DD"
    time: str       # "HH:MM" (24-hr)
    place: str
    language: str   # "en" or "hi"
```

**Response model (`ReadingResponse`):**
```python
class ReadingSection(BaseModel):
    title: str
    icon: str
    content: str

class ReadingResponse(BaseModel):
    sections: list[ReadingSection]
```

### New service: `backend/services/gemini.py`

Responsibilities:
- Read `GEMINI_API_KEY` from environment
- Build the chart summary string from chart data (planets with dignities, houses, ascendant, dasha)
- Construct the prompt (see Prompt Design below)
- Call `genai.GenerativeModel("gemini-2.0-flash").generate_content(prompt)`
- Parse the response into sections using a `===SECTION_NAME===` delimiter
- Raise `HTTPException(503)` if Gemini API is unavailable or key is missing

### Prompt Design

The prompt instructs Gemini to respond in a strict delimited format:

```
You are an expert Vedic astrologer. Based on the following chart data, write a warm, 
insightful reading in [English/Hindi]. Keep each section to 3–5 sentences. 
Be grounded in classical Vedic principles.

CHART DATA:
- Ascendant: Scorpio (14°32', Anuradha nakshatra)
- Moon sign: Cancer (own sign, 9th house)
- Sun: Aries (exalted, 6th house)
- Jupiter: Sagittarius (own sign, 2nd house)
- Saturn: Taurus (7th house, retrograde)
- Mars: Capricorn (exalted, 3rd house)
- Venus: Pisces (exalted, 5th house)
- Mercury: Taurus (6th house)
- Rahu: Gemini (8th house), Ketu: Sagittarius (2nd house)
- Current Dasha: Jupiter MD → Venus AD (ends 2026-03-15)

Write the following 6 sections using exactly these delimiters:

===Personality & Appearance===
[your text]

===Career & Wealth===
[your text]

===Relationships & Marriage===
[your text]

===Health===
[your text]

===Spiritual Inclination===
[your text]

===Current Period (Dasha)===
[your text]
```

### Planet dignity detection (in `gemini.py`)

Before building the prompt, derive each planet's dignity:

| Dignity | Condition |
|---|---|
| Exalted | planet's sign matches classical exaltation sign |
| Debilitated | planet's sign matches classical debilitation sign |
| Own sign | planet's sign is one of its own signs |
| Retrograde | `planet.retrograde == True` |

This enriches the prompt so Gemini can reason about chart strength accurately.

### Configuration

- `GEMINI_API_KEY` stored in `backend/.env` (not committed to git)
- Backend reads it: `api_key = os.getenv("GEMINI_API_KEY")`
- If key is missing: endpoint returns `HTTP 503` with message `"GEMINI_API_KEY not configured"`
- Add `backend/.env` to `.gitignore` if not already present

## Frontend

### `ChartReading.jsx`

Three render states based on component-level state (`idle | loading | done | error`):

**Idle:**
- Centred card with 🔮 icon
- "Vedic Chart Reading" heading + one-line description
- Purple "✨ Generate Reading" button
- Small note: "Powered by Gemini 2.0 Flash · ~5 seconds"

**Loading:**
- ⏳ spinner icon
- "Reading the stars…" text
- Animated progress bar

**Done:**
- Heading "🔮 Your Vedic Reading"
- 6 section cards, each with: icon, title (bold, purple), paragraph text
- "🔄 Regenerate" link (resets to idle)

**Error:**
- Red alert card: "Could not generate reading. Please try again."
- "Try again" button (resets to idle)

**Section icons:**

| Section | Icon |
|---|---|
| Personality & Appearance | 🧬 |
| Career & Wealth | 💼 |
| Relationships & Marriage | 💞 |
| Health | 🌿 |
| Spiritual Inclination | 🕉️ |
| Current Period (Dasha) | ⏳ |

### API call from frontend

```js
const lang = i18n.language.startsWith('hi') ? 'hi' : 'en'
const res = await fetch('/api/kundli/reading', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ ...birthInput, language: lang })
})
const data = await res.json()  // { sections: [...] }
```

`birthInput` is already available in `Result.jsx` as `state.input`.

### `Result.jsx` changes

- Add `'reading'` to `TABS` array at the end: `['birth_chart', 'navamsa', 'dasha', 'planets', 'reading']`
- Render `<ChartReading input={input} />` when `activeTab === 'reading'`
- Pass `input` (date/time/place) as prop; `ChartReading` manages its own fetch

## Error Handling

| Scenario | Backend | Frontend |
|---|---|---|
| `GEMINI_API_KEY` missing | `HTTP 503` | Error state: "Reading service not configured" |
| Gemini API timeout / error | `HTTP 503` | Error state: "Could not generate reading" |
| Place not found | `HTTP 400` | Error state: shows geocode error |
| Malformed Gemini response | Parse what's available; fill missing sections with empty string | Shows partial sections |

- Navamsa-specific reading (Lagna chart only)
- Yoga detection (e.g., Raj Yoga, Dhana Yoga) — future enhancement
